# OSource Global Internal Document Standard

## Mandatory Header Elements
1. **Project Code** — OSource project code (e.g., PROJ001) referenced in the document
2. **Client Name** — Client organisation clearly identified
3. **Document Type** — Document type (BRD / FSD / SRS / User Manual / Product Brochure) stated
4. **Author Information** — Author name or team identified
5. **Confidentiality** — Appropriate confidentiality classification present

## Content Quality
6. **Brand Voice Compliance** — Professional, concise tone consistent with OSource brand guide
7. **No Placeholder Text** — No "TBD", "TODO", or "to be confirmed" left in final sections
8. **Formatting Consistency** — Heading hierarchy (H1 → H2 → H3) consistent throughout
9. **Acronym Definitions** — All acronyms defined on first use

## Business Context
10. **Business Value Statement** — Why this document or project matters to the client stated
11. **Success Definition** — Explicit definition of what project success looks like
12. **Dependencies Identified** — External dependencies and third-party interfaces listed
13. **Next Steps / Action Items** — Clear decisions or actions recorded where applicable

## Version Control
14. **Version Number** — Semantic version (e.g., v1.0) present
15. **Review Date** — Scheduled review or expiry date indicated
