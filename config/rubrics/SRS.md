# Software Requirements Specification (SRS) — Compliance Rubric

## Required Sections
1. **Introduction** — Purpose, scope, and overview of the SRS provided
2. **Overall Description** — Product perspective, functions, user characteristics described
3. **Functional Requirements** — Numbered, uniquely identified functional requirements (FR-xxx)
4. **Non-Functional Requirements** — Performance, reliability, availability, security requirements specified with measurable targets
5. **External Interface Requirements** — UI, hardware, software, and communication interfaces described
6. **System Constraints** — Technical, regulatory, and business constraints documented
7. **Assumptions and Dependencies** — Dependencies on other systems or external factors listed
8. **Glossary** — Technical terms and abbreviations defined

## Quality Criteria
9. **IEEE 830 Alignment** — Requirements follow shall/shall not language convention
10. **Completeness** — No open items marked TBD in final requirement statements
11. **Testability** — Each requirement is objectively verifiable
12. **Traceability Matrix** — Requirements traceable to business objectives or use cases
13. **Priority Classification** — Requirements classified as Must Have / Should Have / Could Have
14. **Version and Change History** — Document version and revision history included
