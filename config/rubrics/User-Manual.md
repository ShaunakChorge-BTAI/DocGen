# User Manual — Compliance Rubric

## Required Sections
1. **Introduction** — Purpose of the manual and target audience stated
2. **Getting Started** — Prerequisites, installation, or initial setup steps provided
3. **System Requirements** — Hardware and software requirements listed
4. **Key Features Overview** — Major features summarised for orientation
5. **Step-by-Step Instructions** — Core workflows explained with numbered steps
6. **Screenshots / Diagrams** — References to visuals or placeholders for UI screenshots present
7. **Troubleshooting** — Common errors and their resolutions documented
8. **FAQ** — Frequently asked questions addressed
9. **Glossary** — Product-specific terms defined
10. **Support / Contact Information** — How to get help clearly indicated

## Quality Criteria
11. **Task-Oriented Structure** — Content organised around user tasks, not system features
12. **Plain Language** — Instructions use simple, jargon-free language
13. **Sequential Clarity** — Steps are numbered and logically ordered
14. **No Placeholder Text** — No "TBD", "TODO", or "see screenshot" without actual reference
15. **Version Reference** — Document version and applicable software version stated
